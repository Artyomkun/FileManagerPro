# === Документация проекта C++ ===

Эта папка содержит документацию для демонстрационного проекта C++.
File Manager Pro позволяет удобно работать с документацией в формате Markdown.

## Содержимое

    1. API.md

    Полная документация API проекта:
        - Описание всех классов и функций
        - Примеры использования
        - Примечания по совместимости

    2. BUILDING.md

    Инструкции по сборке проекта:
        - Требования к системе
        - Сборка через CMake
        - Сборка через Makefile
        - Интеграция с IDE

    3. USAGE.md

    Руководство пользователя:
        - Быстрый старт
        - Примеры использования
        - Конфигурация проекта
        - Отладка и тестирование

    4. CONTRIBUTING.md

    Руководство для контрибьюторов:
        - Стандарты кодирования
        - Процесс внесения изменений
        - Тестирование изменений
        - Создание документации

## Использование с File Manager Pro

    1. Просмотр документации:
        - Откройте файлы .md двойным кликом
        - Используйте предпросмотр Markdown
        - Ищите по всей документации (Ctrl+Shift+F)

    2. Редактирование документации:
        - Встроенный редактор Markdown
        - Подсветка синтаксиса
        - Предпросмотр в реальном времени
        - Проверка ссылок

    3. Навигация по документации:
        - Дерево документации
        - Оглавление файлов
        - Перекрестные ссылки
        - Быстрый поиск

## Форматы документации

    Markdown (.md):
    - Легкий для чтения и редактирования
    - Поддержка таблиц, кода, изображений
    - Конвертируется в HTML, PDF

    Текстовые файлы (.txt):
    - Простые инструкции
    - Быстрый доступ
    - Минимальное форматирование

    Изображения:
    - Диаграммы архитектуры
    - Скриншоты интерфейса
    - Графики и схемы

## Рекомендации по документации

    Для разработчиков:
    - Документируйте публичное API
    - Добавляйте примеры кода
    - Обновляйте при изменении кода

    Для пользователей:
    - Пишите простым языком
    - Добавляйте скриншоты
    - Обеспечьте пошаговые инструкции

    Для контрибьюторов:
    - Описывайте процесс разработки
    - Определяйте стандарты кода
    - Объясняйте архитектуру проекта
